## 1. 网络基础

### 网络的概念

> 网络是将分布在不同地理位置的计算机、终端设备通过通信线路连接起来，按照网络协议进行数据通信，实现资源共享的系统。

**网络的三大要素：**
- 通信主体：计算机、交换机、路由器等设备
- 通信介质：双绞线、光纤、无线电磁波
- 通信协议：TCP/IP、HTTP、OSPF 等规则

### OSI 七层模型

> OSI (Open System Interconnection) 是国际标准化组织提出的网络通信参考模型，将通信过程分为 7 层。

| 层号 | 名称 | 主要功能 | 典型协议/设备 |
|------|------|---------|-------------|
| 7 | **应用层** | 为用户提供网络服务接口 | HTTP, FTP, DNS, SMTP |
| 6 | **表示层** | 数据格式转换、加密/解密 | JPEG, ASCII, SSL |
| 5 | **会话层** | 建立、管理、终止会话 | NetBIOS, RPC |
| 4 | **传输层** | 端到端可靠传输、流量控制 | TCP, UDP |
| 3 | **网络层** | 逻辑寻址、路由选择 | IP, ICMP, 路由器 |
| 2 | **数据链路层** | 物理寻址、差错校验 | MAC, 交换机 |
| 1 | **物理层** | 比特流传输 | 双绞线、光纤、集线器 |

**口诀**：**物数网传会表应** —— 物理层 → 数据链路层 → 网络层 → 传输层 → 会话层 → 表示层 → 应用层

### TCP/IP 四层模型

> TCP/IP 是互联网实际使用的协议栈，将 OSI 七层合并为四层。

| TCP/IP 层 | 对应 OSI 层 | 核心协议 |
|-----------|------------|---------|
| **应用层** | 5~7 层（应用+表示+会话） | HTTP, FTP, DNS, SMTP |
| **传输层** | 4 层（传输层） | **TCP**, **UDP** |
| **网络层** | 3 层（网络层） | **IP**, ICMP, ARP |
| **网络接入层** | 1~2 层（物理+链路） | 以太网, Wi-Fi |

### TCP vs UDP

| | TCP | UDP |
|---|-----|-----|
| **连接方式** | 面向连接（三次握手） | 无连接 |
| **可靠性** | 可靠，确认重传 | 不可靠，不确认 |
| **速度** | 较慢 | 快速 |
| **适用场景** | 网页、文件传输、邮件 | 视频直播、语音通话、DNS |
| **比喻** | 打电话（先接通再说话） | 发短信（直接发，对方收没收到不管） |

### 网络设备在哪层工作

| 设备 | 工作层 | 说明 |
|------|--------|------|
| **集线器 (Hub)** | 物理层 | 信号放大，泛洪转发 |
| **交换机 (Switch)** | 数据链路层 | 基于 MAC 地址转发 |
| **路由器 (Router)** | 网络层 | 基于 IP 地址转发 |
| **防火墙 (Firewall)** | 网络层/传输层 | 安全过滤 |
| **三层交换机** | 网络层+链路层 | 路由 + 交换一体 |

## 2. IP 地址与子网划分

### IP 地址分类

> IP 地址是网络中设备的逻辑地址，IPv4 共 32 位，分为 4 组，每组 8 位，用点分十进制表示。

| 类别 | 首字节范围 | 默认掩码 | 网络数 | 主机数 |
|------|-----------|---------|--------|--------|
| **A 类** | 1 ~ 126 | `255.0.0.0` /8 | 126 | 约 1600 万 |
| **B 类** | 128 ~ 191 | `255.255.0.0` /16 | 约 1.6 万 | 65534 |
| **C 类** | 192 ~ 223 | `255.255.255.0` /24 | 约 200 万 | 254 |
| **D 类** | 224 ~ 239 | — | 组播地址 | — |
| **E 类** | 240 ~ 255 | — | 保留 | — |

> ⚠️ `127.x.x.x` 是**环回地址**（本地测试用）。`0.0.0.0` 表示所有网络，`255.255.255.255` 是广播地址。

### 私有地址范围

> 私有地址只能在内网使用，不能出现在公网上，需要通过 NAT 转换才能访问互联网。

| 类别 | 私有地址范围 |
|------|-------------|
| A 类 | `10.0.0.0 ~ 10.255.255.255` |
| B 类 | `172.16.0.0 ~ 172.31.255.255` |
| C 类 | `192.168.0.0 ~ 192.168.255.255` |

### 子网掩码

> 子网掩码用来区分 IP 地址中哪部分是**网络位**，哪部分是**主机位**。

```
IP 地址：  192.168.  1.  100
子网掩码： 255.255.255.   0
           └─网络位─┘ └主机位┘
```

**子网掩码的两种写法：**
```
255.255.255.0    =  /24   （24 个连续的 1）
255.255.0.0      =  /16
255.0.0.0        =  /8
255.255.255.128  =  /25
```

**速算公式：**
```
可用主机数 = 2^(32 - 掩码位数) - 2    （减去网络地址和广播地址）
子网数     = 2^(借用主机位数)
```

### VLSM 示例

> 将 `192.168.1.0/24` 分成 4 个子网，每个子网约 60 台主机。

```
需要借 2 位主机位作子网位（2² = 4 个子网）
新掩码：/26（24 + 2）

子网 1：192.168.1.0   /26   可用 IP：192.168.1.1 ~ 192.168.1.62
子网 2：192.168.1.64  /26   可用 IP：192.168.1.65 ~ 192.168.1.126
子网 3：192.168.1.128 /26   可用 IP：192.168.1.129 ~ 192.168.1.190
子网 4：192.168.1.192 /26   可用 IP：192.168.1.193 ~ 192.168.1.254
```

## 3. 以太网与交换技术

### MAC 地址

> MAC (Media Access Control) 地址是网络设备的物理地址，48 位，全球唯一。

```
格式：xx:xx:xx:xx:xx:xx（十六进制）
示例：00:e0:fc:12:34:56
前 24 位：厂商代码（OUI）
后 24 位：设备编号
```

### 交换机工作原理

> 交换机基于 **MAC 地址表** 进行转发，学习每个端口连接的设备的 MAC。

**工作流程：**
```
1. 收到数据帧 → 记录源MAC+端口号到MAC地址表
2. 查目标MAC → 如果在表中 → 从对应端口单播转发
3. 如果不在表中 → 泛洪（向所有端口转发，接收端口除外）
```

| 转发方式 | 说明 |
|---------|------|
| **单播** | 已知目标 MAC → 精准转发 |
| **泛洪** | 目标 MAC 未知或为广播 → 所有端口发送 |
| **丢弃** | 目标端口 = 来源端口 → 不转发 |

### VLAN

> VLAN (Virtual LAN) 将一个物理交换机**逻辑划分为多个独立的广播域**。不同 VLAN 之间默认不能通信，需要三层设备（路由器或三层交换机）。

```python
VLAN 的好处：
  隔离广播域 —— 减少广播风暴
  提高安全性 —— 不同部门互不可见
  灵活组网 —— 物理位置无关，同一 VLAN 可跨交换机
```

**VLAN 接口类型：**

| 类型 | 说明 | 场景 |
|------|------|------|
| **Access** | 只属于一个 VLAN | 连接 PC、服务器 |
| **Trunk** | 承载多个 VLAN（打标签） | 交换机与交换机之间 |
| **Hybrid** | 华为特有，灵活控制 | 混合场景 |

**VLAN ID 范围：**
- 0 ~ 4095（标准 1 ~ 4094 可用）
- VLAN 1 是默认 VLAN

### STP 生成树协议

> STP (Spanning Tree Protocol) 通过阻塞冗余链路来**防止二层环路**，同时提供链路备份。

```
无 STP：广播帧在环路中无限循环 → 广播风暴 → 网络瘫痪
有 STP：自动阻塞冗余端口 → "树"形拓扑 → 无环路
```

## 4. 路由技术

### 路由原理

> 路由是**选择数据包从源到目的地的最佳路径**的过程。路由器根据路由表进行转发决策。

**路由表包含的信息：**
```
目的网络地址 + 子网掩码 → 下一跳地址 + 出接口
```

### 路由优先级与度量值

| 路由类型 | 优先级 | 说明 |
|---------|--------|------|
| **直连路由** | 0 | 路由器直接连接的网络 |
| **静态路由** | 60 | 手动配置 |
| **OSPF** | 10（内部）/ 150（外部） | 动态路由协议 |
| **RIP** | 100 | 距离矢量协议 |

> 数字越小优先。优先级相同时比较度量值（Cost）。

### 静态路由

> 手动配置路由表项，适用于小型稳定网络。

```
ip route-static 目标网络 掩码 下一跳地址
ip route-static 192.168.2.0 255.255.255.0 10.0.0.2
```

### OSPF 基础

> OSPF (Open Shortest Path First) 是链路状态路由协议，适用于中大型网络。

| 概念 | 说明 |
|------|------|
| **Router ID** | 路由器唯一标识 |
| **Area** | 区域，Area 0 是骨干区域 |
| **Cost** | 开销 = 参考带宽 / 接口带宽 |
| **邻居** | 直连的 OSPF 路由器 |
| **LSA** | 链路状态通告，描述网络拓扑 |

### 路由选路原则

```
1. 最长掩码匹配（最精确的路由优先）
2. 优先级小的优先
3. 度量值小的优先
```

## 5. 网络服务与应用

### DHCP

> DHCP (Dynamic Host Configuration Protocol) 动态分配 IP 地址等网络参数。

**工作流程（四次握手）：**
```
客户端 → 服务器：DHCP Discover   （发现——找 DHCP 服务器）
服务器 → 客户端：DHCP Offer      （提供——分配一个 IP）
客户端 → 服务器：DHCP Request    （请求——确认要这个 IP）
服务器 → 客户端：DHCP ACK        （确认——给你用）
```

### DNS

> DNS (Domain Name System) 将域名解析为 IP 地址。

```
www.huawei.com → DNS 查询 → 119.3.30.36
```

### NAT

> NAT (Network Address Translation) 将**私有地址转换为公网地址**，让内网设备访问互联网。

| 类型 | 说明 |
|------|------|
| **静态 NAT** | 一对一映射（内网→固定公网 IP） |
| **动态 NAT** | 内网→公网地址池中随机一个 |
| **NAPT/PAT** | 多内网 IP + 端口 → 一个公网 IP + 不同端口（最常用） |

### ACL 访问控制列表

> ACL 通过规则过滤流量，实现网络安全控制。

```
基本 ACL：只根据源 IP 过滤（编号 2000~2999）
高级 ACL：根据源 IP、目标 IP、端口、协议过滤（编号 3000~3999）
```

## 6. 华为设备基础

### VRP 系统

> VRP (Versatile Routing Platform) 是华为设备的操作系统。

### 命令行视图

```bash
用户视图     <Huawei>               # 查看简单信息
系统视图     [Huawei]               # system-view 进入，配置全局参数
接口视图     [Huawei-GigabitEthernet0/0/1]  # 配置接口
路由协议视图 [Huawei-ospf-1]        # 配置路由协议
```

### 基本配置命令

```bash
# 设备命名
<Huawei> system-view
[Huawei] sysname Core-SW

# 查看信息
<Huawei> display version           # 系统版本
<Huawei> display current-configuration  # 当前配置
<Huawei> display ip interface brief     # 接口 IP 摘要
<Huawei> display vlan                  # VLAN 信息

# 接口配置
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] ip address 192.168.1.1 24
[Huawei-GigabitEthernet0/0/1] undo shutdown    # 开启接口

# VLAN 配置
[Huawei] vlan 10                    # 创建 VLAN 10
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] port link-type access
[Huawei-GigabitEthernet0/0/1] port default vlan 10

# Trunk 配置
[Huawei] interface GigabitEthernet 0/0/24
[Huawei-GigabitEthernet0/0/24] port link-type trunk
[Huawei-GigabitEthernet0/0/24] port trunk allow-pass vlan 10 20

# 静态路由
[Huawei] ip route-static 192.168.2.0 24 10.0.0.2

# 保存配置
<Huawei> save
```

### OSI 七层速记

```
应用层 → HTTP, DNS, FTP           "应"
表示层 → 加密/格式转换               "表"
会话层 → 对话管理                   "会"
传输层 → TCP/UDP                   "传"
网络层 → IP, 路由器                 "网"
链路层 → MAC, 交换机                "链"
物理层 → 线缆, 集线器               "物"
```

### 常见面试题

**1. TCP 三次握手过程？**
> ①客户端→SYN→服务器  ②服务器→SYN+ACK→客户端  ③客户端→ACK→服务器。握手完成后建立可靠连接。

**2. VLAN 的作用？**
> 隔离广播域、提高安全性、灵活组网。不同 VLAN 必须通过三层设备互通。

**3. 静态路由 vs 动态路由？**
> 静态路由手动配置，适合小型网络；动态路由（OSPF、RIP）自动学习，适合中大型网络。

**4. NAT 的作用？**
> 将私有 IP 转换为公网 IP，解决 IPv4 地址不足问题，同时隐藏内网结构提升安全性。

**5. 交换机 vs 路由器？**
> 交换机工作在数据链路层（MAC 转发），路由器工作在网络层（IP 转发）。交换机连接同网段设备，路由器连接不同网段。

**6. 什么是广播域和冲突域？**
> 广播域：能收到同一广播帧的设备范围（路由器隔离广播域）。冲突域：同一时刻只能有一个设备发送数据的范围（交换机每个端口一个冲突域）。
